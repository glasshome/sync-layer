# Plan 01-03: View Memoization + Memo Equality Comparators

## Objective

Prevent redundant object allocation in `buildEntityView` and `buildAreaView`. Add structural equality comparators to `createMemo` calls in hooks so downstream components don't re-render on unchanged data.

## must_haves

- `buildEntityView` does not create new `Date` objects when timestamps are unchanged
- `useEntity()` memo uses equality comparator that checks entity state/attributes
- `buildAreaView` does not iterate all devices/registry when called for an unrelated entity change
- `useAreas()` memo uses equality comparator that prevents re-render on unchanged area data

## Tasks

<task id="1">
<title>Keep timestamps as ISO strings in EntityView and add equality comparator</title>

<read_first>
- packages/public/sync-layer/src/entities/views.ts (full file)
- packages/public/sync-layer/src/core/types.ts (EntityView type definition)
- packages/public/sync-layer/src/solid/hooks.ts (lines 47-62 for useEntity, lines 159-171 for useAreas)
</read_first>

<action>
1. In `types.ts`, change `EntityView` type:
   ```typescript
   // Change from:
   lastChanged: Date;
   lastUpdated: Date;
   // To:
   lastChanged: string;
   lastUpdated: string;
   ```

2. In `views.ts` `buildEntityView()`, stop creating Date objects:
   ```typescript
   // Change from:
   lastChanged: new Date(entity.last_changed),
   lastUpdated: new Date(entity.last_updated),
   // To:
   lastChanged: entity.last_changed,
   lastUpdated: entity.last_updated,
   ```

3. Create an `entityViewEquals` function in `views.ts`:
   ```typescript
   export function entityViewEquals(a: EntityView | undefined, b: EntityView | undefined): boolean {
     if (a === b) return true;
     if (!a || !b) return false;
     return (
       a.id === b.id &&
       a.state === b.state &&
       a.lastChanged === b.lastChanged &&
       a.lastUpdated === b.lastUpdated &&
       a.name === b.name &&
       a.areaId === b.areaId &&
       a.isDisabled === b.isDisabled &&
       a.isHidden === b.isHidden &&
       a.icon === b.icon
     );
   }
   ```

4. In `hooks.ts` `useEntity()` (line 57), add the equality comparator:
   ```typescript
   // Change from:
   return createMemo(() => getEntityView(id));
   // To:
   return createMemo(() => getEntityView(id), undefined, { equals: entityViewEquals });
   ```

5. In `hooks.ts` `useEntities()` (line 78), add an array-level equality comparator:
   ```typescript
   return createMemo(() => {
     return entityIds().map(...).filter(...);
   }, undefined, {
     equals: (a, b) => {
       if (a.length !== b.length) return false;
       return a.every((v, i) => entityViewEquals(v, b[i]));
     }
   });
   ```

6. Grep across `apps/dash` for `.lastChanged.` and `.lastUpdated.` to find any code that calls Date methods on these fields (e.g., `.getTime()`, `.toLocaleString()`). Update those call sites to work with ISO strings instead.
</action>

<acceptance_criteria>
- `types.ts` EntityView has `lastChanged: string` and `lastUpdated: string` (not `Date`)
- `views.ts` `buildEntityView` does NOT contain `new Date(entity.last_changed)` or `new Date(entity.last_updated)`
- `views.ts` exports `entityViewEquals` function
- `hooks.ts` `useEntity` createMemo has `{ equals: entityViewEquals }` option
- No compile errors from Date→string change (check with `bun run build` in sync-layer)
</acceptance_criteria>
</task>

<task id="2">
<title>Optimize buildAreaView and add area memo equality</title>

<read_first>
- packages/public/sync-layer/src/entities/area-views.ts (full file)
- packages/public/sync-layer/src/solid/hooks.ts (lines 159-202 for useAreas/useArea)
</read_first>

<action>
1. In `area-views.ts`, optimize `buildAreaView` to avoid full device/registry scans:
   - Instead of `Object.values(state.devices)` filtering all devices, build a lookup from `state.entityRegistry` entries that have the target `area_id` or whose `device_id` maps to a device in that area.
   - Use targeted property access `state.areas[areaId]` instead of iterating all areas.

2. Create an `areaViewEquals` function:
   ```typescript
   export function areaViewEquals(a: AreaView | undefined, b: AreaView | undefined): boolean {
     if (a === b) return true;
     if (!a || !b) return false;
     return (
       a.id === b.id &&
       a.name === b.name &&
       a.floorId === b.floorId &&
       a.entityCount === b.entityCount
     );
   }
   ```
   (Include `entityCount` — a new derived field — or compare `a.entities.length === b.entities.length` if AreaView has an entities array)

3. In `hooks.ts` `useArea()` (line 181), add equality comparator:
   ```typescript
   return createMemo(() => buildAreaView(id), undefined, { equals: areaViewEquals });
   ```

4. In `hooks.ts` `useAreas()` (line 160), add array-level equality:
   ```typescript
   return createMemo(() => {
     return Object.keys(state.areas)
       .map((areaId) => { try { return buildAreaView(areaId); } catch { return undefined; } })
       .filter((v): v is AreaView => v !== undefined);
   }, undefined, {
     equals: (a, b) => {
       if (a.length !== b.length) return false;
       return a.every((v, i) => areaViewEquals(v, b[i]));
     }
   });
   ```
</action>

<acceptance_criteria>
- `area-views.ts` does NOT use `Object.values(state.devices)` to scan all devices (or scan is replaced with targeted access)
- `area-views.ts` exports `areaViewEquals` function
- `hooks.ts` `useArea` createMemo has `{ equals: areaViewEquals }` option
- `hooks.ts` `useAreas` createMemo has array equality comparator
</acceptance_criteria>
</task>
