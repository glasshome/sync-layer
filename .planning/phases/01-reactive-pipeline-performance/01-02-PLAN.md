# Plan 01-02: Gesture Handler Layout Cache

## Objective

Eliminate `getBoundingClientRect()` calls from pointer event hot paths. Cache element dimensions and invalidate via ResizeObserver.

## must_haves

- Zero `getBoundingClientRect` calls in `onPointerMove` and `onPointerEnter` handlers
- Cached rect invalidated when element resizes
- Slider interaction responsiveness restored

## Tasks

<task id="1">
<title>Replace getBoundingClientRect with cached dimensions in gesture handlers</title>

<read_first>
- packages/public/widget-sdk/src/framework/gestures/use-widget-gestures.ts (full file, focus on lines 110-120, 280-300, and the state object)
</read_first>

<action>
1. Add a cached rect to the gesture state object:
   ```typescript
   let cachedRect: DOMRect | null = null;
   let resizeObserver: ResizeObserver | null = null;
   ```

2. Create a function to get or compute the rect:
   ```typescript
   function getCachedRect(el: HTMLElement): DOMRect {
     if (!cachedRect) {
       cachedRect = el.getBoundingClientRect();
     }
     return cachedRect;
   }
   ```

3. In `onPointerDown` (or wherever the element is first captured), set up a ResizeObserver to invalidate the cache:
   ```typescript
   function observeElement(el: HTMLElement): void {
     if (resizeObserver) resizeObserver.disconnect();
     resizeObserver = new ResizeObserver(() => {
       cachedRect = null;
     });
     resizeObserver.observe(el);
     cachedRect = null; // force fresh measurement on first use
   }
   ```

4. Call `observeElement(el)` in `onPointerEnter` when `state.element` changes (i.e., when entering a new widget element).

5. Replace all `el.getBoundingClientRect()` calls in:
   - `getSlideOrientation()` (around line 115) → use `getCachedRect(el)`
   - `getCursorForElement()` (around line 288) → use `getCachedRect(el)`
   - Any other hot-path uses inside pointer move/enter handlers

6. In `dispose()`, disconnect the ResizeObserver:
   ```typescript
   if (resizeObserver) {
     resizeObserver.disconnect();
     resizeObserver = null;
   }
   cachedRect = null;
   ```

7. Also invalidate cache on scroll events (scroll changes bounding rect position):
   ```typescript
   // In observeElement or setup
   const scrollHandler = () => { cachedRect = null; };
   window.addEventListener("scroll", scrollHandler, { passive: true });
   // Clean up in dispose
   window.removeEventListener("scroll", scrollHandler);
   ```
</action>

<acceptance_criteria>
- `getSlideOrientation` does NOT contain `el.getBoundingClientRect()` — uses `getCachedRect(el)` instead
- `getCursorForElement` does NOT contain `el.getBoundingClientRect()` — uses `getCachedRect(el)` instead
- File contains `new ResizeObserver` for cache invalidation
- `dispose()` function disconnects ResizeObserver
- `cachedRect = null` appears in ResizeObserver callback
</acceptance_criteria>
</task>
